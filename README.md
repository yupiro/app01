# 株価予測アプリ

Yahoo Finance（yfinance）から取得した株価データをもとに、複数のアルゴリズム（LSTM・線形回帰・移動平均トレンド）で将来の株価を予測し、比較・検証できるFlask製Webアプリです。

## 主な機能

- **複数アルゴリズムでの予測比較**: LSTM（深層学習）、線形回帰、移動平均・トレンドの3手法を同時に実行し、1つのチャートと比較表で結果を見比べられます。
- **GAFAM ワンクリック選択**: Google / Apple / Meta / Amazon / Microsoft をボタン一発で予測。
- **GAFAM比較チャート**: GAFAM5銘柄の株価を起点=100として正規化し、1つのチャートで相対パフォーマンスを比較できます（期間：3ヶ月〜5年）。
- **直近＋予測の拡大表示**: 全体チャートでは見えづらい直近期間と予測部分だけを拡大表示し、翌営業日（または次の1本）の予測値をアルゴリズムごとにハイライト表示します。
- **過去日付での予測精度検証（バックテスト）**: 過去の任意の日付を選ぶと、その「予測日数」分前の時点だけのデータで当時の予測をシミュレーションし、実際の価格とのずれ（誤差・誤差率）を検証できます。
- **超短期（分足）予測**: 日足に加え、1分足・5分足・15分足・30分足・1時間足のイントラデイ予測にも対応しています。

## 必要環境

- Python **3.10**（TensorFlowが未対応のため、3.13/3.14などの新しいバージョンでは動作しません）
- インターネット接続（Yahoo Financeからのデータ取得に必要）

## セットアップ

```bash
# 仮想環境の作成（Python 3.10を使用）
py -3.10 -m venv venv

# 依存パッケージのインストール
./venv/Scripts/python.exe -m pip install --upgrade pip
./venv/Scripts/python.exe -m pip install -r requirements.txt
```

## 起動方法

```bash
./venv/Scripts/python.exe app.py
```

起動後、ブラウザで [http://127.0.0.1:5000](http://127.0.0.1:5000) を開いてください。

## 使い方

### 1. 株価を予測する

1. 「銘柄コード」に予測したい銘柄（例: `AAPL`、日本株なら `7203.T` のように末尾に `.T` を付ける）を入力するか、GAFAMボタンをクリックします。
2. 「時間軸」で日足（長期）か、分足・時間足（超短期）かを選びます。
3. 「予測日数」（超短期の場合は「予測本数」）と、使用する「アルゴリズム」（複数選択可）を選びます。
4. 「予測する」をクリックすると、実績データと各アルゴリズムの予測結果が1つのチャートに表示されます。
5. グラフ下の「直近＋予測（拡大表示）」で、直近期間と予測部分だけを拡大したチャートと、次の予測値のバッジを確認できます。
6. 比較表で、各アルゴリズムの最終予測日の価格と騰落率を確認できます。

### 2. GAFAM比較チャートを見る

上部の「GAFAM比較チャート」ボタンをクリックすると、GAFAM5銘柄を起点100に正規化したチャートが表示されます。右上のセレクトボックスで期間（3ヶ月〜5年）を切り替えられます。

### 3. 過去日付での予測精度を検証する（バックテスト）

1. まず銘柄を1回予測してください（バックテストは同じ銘柄・予測日数・アルゴリズムの設定を使います）。
2. 画面下部の「過去日付での予測精度を検証」で、検証したい過去の日付を選びます。
3. 「検証する」をクリックすると、「選択日 − 予測日数」の時点までのデータだけを使って当時の予測をシミュレーションし、実際の値動きと重ねて表示します。誤差・誤差率も表で確認できます（このバックテストは日足データのみに対応しています）。

## アルゴリズムについて

| アルゴリズム | 説明 |
| --- | --- |
| LSTM（深層学習） | 直近60本の値動きを学習し、次の値を逐次予測するニューラルネットワーク |
| 線形回帰 | 直近120本の値動きに直線を当てはめ、延長して予測 |
| 移動平均・トレンド | 直近20本の平均変化量（ドリフト）を最終値から積み上げて予測 |

## 免責事項

このアプリの予測結果は過去データに基づく統計的モデルによる参考値であり、将来の株価を保証するものではありません。投資判断は自己責任で行ってください。
